- [x] [CFG-001] [HIGH] Content duplication between agent and loaded skill - `.claude/agents/claude-config-reviewer.md`
  Issue: The agent's Review Checklists (lines 22-102) substantially restate conventions from
  `claude-config-conventions` SKILL.md. For example, the Agent Frontmatter checklist (lines 27-31)
  mirrors the skill's "Agent Conventions > Frontmatter" section (lines 25-29), and the Architecture
  Principles section (lines 99-102) restates "Content Placement Rules" (lines 17-21). The architecture
  principle is: skills define WHAT, agents define HOW. The checklists sit in a gray area -- they are
  formatted as workflow, but their content is the same principles from the skill.
  Fix: Consider referencing the skill's sections rather than restating them. The agent could say
  "Verify agent frontmatter against conventions in claude-config-conventions skill" rather than
  re-listing each field. Alternatively, keep the checklists but accept this as intentional
  operationalization (reviewer checklists are inherently derived from conventions).

- [x] [CFG-002] [HIGH] Missing report persistence instruction - `.claude/agents/claude-config-reviewer.md`
  Issue: The command `.claude/commands/review-claude-config.md` (line 12) states the subagent will
  "Persist report to `docs/config-reviews/` if issues found", but the agent file itself contains no
  instruction to write output to any file. The agent only defines an Output Format for conversation
  display. The subagent will not persist reports unless explicitly instructed.
  Fix: Add a section to the agent (e.g., after "After Review") instructing it to write the review
  report to `docs/config-reviews/YYYY-MM-DD-context.md` when issues are found.

- [x] [CFG-003] [HIGH] Stale command reference `/commit` in new file - `.claude/commands/review-claude-config.md`
  Issue: Next Commands section (line 20) references `/commit` but no `commit.md` exists in
  `.claude/commands/`. The actual command is `/git-commit` (file: `git-commit.md`). While this
  reference also exists pre-change in `update-claude-config.md` and `CLAUDE.md`, this new file
  propagates the same broken reference.
  Fix: Change `/commit` to `/git-commit` in `review-claude-config.md`. Also fix the same reference
  in `update-claude-config.md` (line 26), `claude-config-updater.md` (line 71), and `CLAUDE.md`
  (line 51) for consistency. Note: MEMORY.md suggests `/commit` is VCS-agnostic by design -- if
  that's intentional, create a `commit.md` command that delegates to jj or git.
